# Compatibility support for 'RAVE' 1.0 format

Convert 'RAVE' subject generated by 2.0 pipeline such that 1.0 modules
can use the data. The subject must have valid electrodes. The data must
be imported, with time-frequency transformed to pass the validation
before converting.

## Usage

``` r
rave_subject_format_conversion(subject, verbose = TRUE, ...)
```

## Arguments

- subject:

  'RAVE' subject characters, such as `'demo/YAB'`, or a subject instance
  generated from [`RAVESubject`](RAVESubject.md)

- verbose:

  whether to verbose the messages

- ...:

  ignored, reserved for future use

## Value

Nothing
